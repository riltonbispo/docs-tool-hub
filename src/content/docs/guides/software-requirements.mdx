---
  title: Requisitos Funcionais e Não Funcionais
---

import { Tabs, TabItem } from '@astrojs/starlight/components';

<Tabs>
  <TabItem label="Funcionais" icon="pencil">
    1. **Separar Ferramentas por Escopos (RF1)**
        - A plataforma deve permitir a categorização de ferramentas por diferentes escopos de uso.

    2. **Pesquisa de Ferramentas (RF2)**
        - O usuário deve poder realizar buscas por ferramentas específicas utilizando uma barra de pesquisa.

    3. **Link para a Documentação da Ferramenta (RF3)**
        - Cada ferramenta cadastrada na plataforma deve incluir um link direto para sua documentação oficial.

    4. **Múltiplos Idiomas (RF4)**
        - A plataforma deve oferecer suporte a múltiplos idiomas, permitindo ao usuário selecionar o idioma de preferência.

    5. **Descrição das Ferramentas (RF5)**
        - A plataforma deve exibir uma descrição detalhada de cada ferramenta cadastrada.
  </TabItem>
  <TabItem label="Não Funcionais" icon="pencil">
    1. **Desempenho (RNF1)**
        - A plataforma deve garantir um bom desempenho, mesmo com um grande número de usuários simultâneos e ferramentas cadastradas.

    2. **Acessibilidade e Responsividade (RNF2)**
        - A plataforma deve ser acessível e responsiva, proporcionando uma boa experiência de usuário em diversos dispositivos e para pessoas com necessidades especiais.
  </TabItem>

</Tabs>

---

Esta documentação lista os principais requisitos funcionais e não funcionais que a plataforma deve atender para garantir seu sucesso e qualidade.
